# Carelane Pilot Facilitation Script v1

**Purpose:** concise facilitation script for the first controlled closed-pilot session.
**Rule:** this is a controlled pilot, not production.

## 1. Session opening

**Facilitator says**

- “We are here to observe how Carelane supports a controlled closed pilot.”
- “Carelane is a workflow and coordination layer for anonymous or pseudonymous youth-care matching and throughflow.”
- “Carelane is not an ECD, not a municipal ERP, and not a permanent dossier platform.”
- “This is not production.”
- “We expect the session to take about 30 to 45 minutes, depending on questions.”
- “I will capture feedback in the pilot feedback log, and we will separate blockers from observations.”

**Participant sees**

- a controlled pilot environment
- seeded pilot/demo data
- a canonical workflow with the current operational context

**Question to ask**

- “Before we start: what do you think this system is trying to help you do?”

**Observation to capture**

- first impression of purpose
- whether the participant understands this is a pilot, not production
- any immediate confusion about scope or ownership

## 2. Demo storyline

### Aanmelding

**Facilitator says**

- “We start with Aanmelding.”
- “This is where the case enters the operational flow.”

**Participant sees**

- the casus / intake context
- the current state and next step
- the action that moves the case forward

**Question to ask**

- “What do you think is happening here?”

**Observation to capture**

- whether the participant can identify the current state
- whether the next step is obvious
- any missing context or unclear labels

### Matching

**Facilitator says**

- “Now we start Matching.”
- “Matching remains advisory, not automatic assignment.”
- “We expect to see the recommendation and why it was suggested.”

**Participant sees**

- advisory match context
- explanation or trade-offs where applicable
- the current worklist or matching queue

**Question to ask**

- “What information do you need to trust this step?”

**Observation to capture**

- whether the recommendation is understandable
- whether trade-offs are visible
- whether the participant treats matching as advice or assignment

### Aanbiederreactie

**Facilitator says**

- “Now the provider response is processed.”
- “Only the linked provider should see this casus in their queue.”

**Participant sees**

- provider-facing review context
- case-linked visibility
- the response action or decision path

**Question to ask**

- “What would you do next?”

**Observation to capture**

- whether provider visibility is appropriate
- whether the participant expects an approved link before access
- any permission concerns

### Plaatsing

**Facilitator says**

- “We confirm the placement step.”
- “This is the point where the casus moves beyond response into placement handling.”

**Participant sees**

- placement confirmation context
- state transition evidence
- the current action or status update

**Question to ask**

- “Where would this fit or not fit in your current process?”

**Observation to capture**

- whether placement status is clear
- whether the participant understands ownership at this step
- any process mismatch or terminology issue

### Intake

**Facilitator says**

- “Finally, we move toward Intake.”
- “This follows placement and should not appear before it.”

**Participant sees**

- intake-ready or intake-start context
- the final handoff state
- the visible transition trail

**Question to ask**

- “What feels unclear, risky, or unnecessary?”

**Observation to capture**

- whether the transition order feels correct
- whether intake appears only after placement
- whether anything seems premature or out of sequence

## 3. Participant prompts

Use these prompts during the walkthrough:

- “What do you think is happening here?”
- “What would you do next?”
- “What information do you need to trust this step?”
- “Where would this fit or not fit in your current process?”
- “What feels unclear, risky, or unnecessary?”

## 4. Trust and safety checks

- confirm role-based visibility for the current participant
- verify provider visibility only after an approved link or request path
- verify audit trail visibility for meaningful transitions
- repeat that this is not production
- confirm that no real personal data is used unless formally approved
- stop immediately if a permission leak or workflow break is observed

## 5. Closing

**Facilitator says**

- “Let’s recap what we saw.”
- “What are the most important observations?”
- “Should we continue, pause, or adjust the pilot?”

**Recap questions**

- “What worked well?”
- “What was unclear?”
- “What felt risky?”
- “What would you change before the next session?”

**Capture**

- top 3 observations
- top 3 fixes
- owner / action list
- decision: continue, pause, or adjust pilot
